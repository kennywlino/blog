Intro

These days, when people start learning how to program, they often start with Python. That's probably because it's known for being one of the easier programming languages to pick up. In fact, the PopularitY of Programming Language Index (PYPL) says that Python is the world's most popular language!

Even though Python is a valuable Swiss Army knife, there are actually a number of tools available through the Terminal. But what's the Terminal? When you put it that way, it sounds kind of scary...

Short background on Unix

The DEC VT100, an old computer terminal (from Wikipedia)

If you open up the Terminal, you might see something that looks like the screen here. You might be thinking that this looks a lot like the screens hackers use-- which is actually more or less right!

You see, almost all operating systems (OS) - Microsoft's systems are based off of a system called Unix. Unix was born in the early 70s at Bell Labs, which was a part of the Bell System.

The Bell System represented the group of companies that were owned by the Bell Telephone Company which became what we know today as the American Telephone and Telegraph Company, a.k.a. AT&T! If you're interested in knowing more about pre-modern day tech monopolies, I would recommend reading about them

GUI :--: TERMINAL

Before we got to clicking around on our screens, we had to navigate using the terminal. When icons and menus -- or formally, the graphical user interface (GUI) -- came around, that made using the terminal obsolete for the everyday consumer. Or at least the type of people who spend most of their time browsing the Internet and watching videos :woozy_face:

If you want to know more about Unix and its origins, check out this article from HowToGeek or everybody's favorite, Wikipedia.

Unix for Poets

Now that we know how to do some basic navigation through the terminal, let's experiment with some of the pre-installed tools.

Here, we'll work with a text file to extract basic info like the total count of words, finding commonly co-occurring words (n-grams), and applying filters.

To do this, we'll be following info taken from Unix for Poets. Unix for Poets is a well-known document by Kenneth Ward Church that describes how easy it is to do some of this basic text processing right in the command line.

I actually was first introduced to this during my master's program and I've seen it appear in other lecturer's resources as well, so I hope this proves to be just as useful for you!

The next step we want to do in order to count the words is to actually separate them. What do I mean by that? Well, we need a way for our tools to distinguish one word from another. The easiest way here is to put each word on its own line.

To do this in bash, we'll use the translate tool, or tr to replace all the whitespace with a newline character. Because there are a lot of possible whitespace characters, we'll take the opposite or, the complement of anything that's not A-Z.

! tr -sc '[A-Z][a-z]' '\n' < ../../01_the_ugly_duckling.txt
